Gen IV Switch-In Calculator

Models the Platinum trainer AI's full send-in logic: Phase 1 (guaranteed switch to the first Pokémon, by type score order, that actually knows a supereffective move), Phase 2 (if none do, whichever Pokémon's moves deal the most calculated damage), and Phase 3 (fallback to the first Pokémon in the party that can switch in at all).

Switch-In Result

Phase 1 (type): party sorted by type-matchup score (4x=160, 2x=80, 1x=40, 1/2x=20, 1/4x=10, 0x=0; single type scored twice; 320 total overflows to 64). The AI then checks each Pokémon in that order and locks in the first one that knows any move super-effective (2x+) against your type(s) — status moves count here too, only their type matters, not whether they deal damage.
Phase 2 (damage): if nobody qualifies in Phase 1, every damaging move on every Pokémon is calculated (simplified: (2×Level/5+2)×Power×Atk/Def/50+2, ×STAB, ×type effectiveness, no crit/random roll/items/weather) and the single highest-damage Pokémon is sent in. Matching the documented Gen4 bug, damage values wrap back to 0 past 255. A specific list of moves is excluded from this calculation entirely, per the documented rules — Explosion, Self-Destruct, Focus Punch, Superpower, Sucker Punch, Hyper Beam, Frenzy Plant, Blast Burn, Hydro Cannon, Giga Impact, Solar Beam, Eruption, Water Spout, Head Smash, and a handful of others the AI's own damage check simply doesn't count, even though they deal real damage in an actual fight. A Pokémon whose only damaging moves are on this list is treated as having none for Phase 2 purposes.
Target ability: if your active Pokémon has an immunity ability (Levitate, Volt Absorb, Water Absorb, Flash Fire, etc.), select it — this correctly nullifies the matching move type for both the Phase 1 supereffective check and the Phase 2 damage calc, matching the source doc's note that the switching AI's type check accounts for abilities like Levitate. (Phase 1's underlying type-score sort deliberately ignores abilities, per the same doc — only the "does it have a supereffective move" step and the damage calc use ability corrections.)
Held items & weather: per the documented AI rules, the switching AI's damage check does apply certain "input" multipliers when judging candidates — Choice Band/Specs, type-boosting held items (Charcoal, Mystic Water, etc.), species-locked items (Light Ball, Thick Club), and weather boosts to Fire/Water moves. These are now factored into Phase 2. Life Orb, Metronome, and Expert Belt are deliberately left out — the documented AI logic explicitly ignores these three for this specific check, so omitting them is accurate, not a gap. On your side, held items are mostly irrelevant here too (the AI ignores damage-reducing berries for this check) — the one exception is Iron Ball, which removes Flying/Levitate's Ground immunity and is modeled accordingly.
Weather source: weather here means the actual field weather (from a move, ability, or item already in play), which the tool doesn't track automatically — set it manually if applicable.
Gravity: per the same source doc, Gravity is one of the field effects the switching AI's type check accounts for — while active, it grounds Flying-types and Levitate holders, so Ground-type moves stop being blocked by either. Toggle it on if it's been used earlier in the fight (it lasts 5 turns and persists through switches).
Light Screen / Reflect: the same "input multiplier" list in the source doc explicitly names these — they halve incoming Special/Physical damage respectively for the side they're set on. These are set per-side (your whole team), not per-Pokémon, so there's one toggle each rather than one per target.
Opponent abilities: each opponent Pokémon now has an editable Ability field (auto-filled from the trainer database). Huge Power/Pure Power correctly double their effective Attack for physical moves. Thick Fat is offered as a target-ability option since it halves incoming Fire/Ice damage. Torrent/Blaze/Overgrow/Swarm (1.5x same-type move under 1/3 HP) are not modeled, since they depend on current HP, which this tool doesn't track.
Doubles mode: the source docs only define the switch-in phases for a single target, so this is an extension, not documented Gen4 behavior. With two active Pokémon, the type score is the average of the score against each target, the supereffective check passes if either target is hit for 2x+, and known spread moves (Earthquake, Surf, Discharge, Rock Slide, Heat Wave, Lava Plume, Muddy Water, Magnitude, Explosion/Self-Destruct) get the documented 0.75x multi-target damage reduction. The separate "Tag Strategy" AI flag (partner-synergy logic — not Earthquaking an ally with Levitate, not Discharging a Water-type partner, etc.) governs which move she picks mid-fight, not which Pokémon gets sent in, so it isn't modeled here.
Phase 3 (fallback): if no Pokémon has a damaging move that connects at all, the first living Pokémon in party order is sent in.
This is an approximation for planning purposes — abilities/items affecting damage and weather are now modeled (see below); crit hits, random damage rolls, and a handful of nonstandard moves (Counter, Endeavor, etc.) still aren't.

Attacker-side abilities now modeled: Normalize (all the candidate's moves count as Normal-type for this check), Scrappy (Normal/Fighting moves stop being blocked by Ghost's immunity, treated as neutral rather than supereffective), Mold Breaker (ignores every target-side immunity ability, e.g. Levitate/Wonder Guard/Volt Absorb, for that candidate's moves), and Adaptability (STAB becomes 2x instead of 1.5x). All four are read from each opponent's Ability field, same as Huge Power/Thick Fat.
Dual non-immunity glitch: per the documented bug, if your active Pokémon's first-listed type is immune to a move and its second-listed type is weak to it, the real AI's supereffective check incorrectly treats it as supereffective anyway. This is now replicated for Phase 1's check specifically (not the type-score sort or Phase 2 damage, which the doc doesn't say are affected).

Trainer auto-fill (all 483 trainers with a set roster): stats, levels, natures, and movesets are computed directly from the Platinum Kaizo project's own data (party order, sets, species, and move tables) — Atk/SpA from base stats + IVs + level + nature (EVs are 0, per the source). Move base powers reflect Kaizo's own rebalanced values (many are buffed well above vanilla Platinum), not vanilla Gen 4 values. Hidden Power's power is approximated at 70 since it depends on IVs not encoded in the sets data; recoil/fixed-damage moves like Counter keep their real type for the Phase 1 check but aren't modeled in the Phase 2 damage calc. Type the trainer's exact name (as it appears in-game, e.g. "Leader Fantina") and pick the suggestion, or hit Load.
